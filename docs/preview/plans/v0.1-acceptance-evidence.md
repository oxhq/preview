## Current Proof

- Full suite: 141 tests, 606 assertions on PHP 8.5.5.
- Generic, HMAC, and Stripe capture-to-fixture-to-test flows are covered by package-internal Testbench acceptance tests.
- GitHub and Shopify providers pass the shared provider contract for signature verification, event extraction, fixture generation, test generation, and fresh signing.
- Exact replay preserves local raw headers and body while show/fixture metadata remains redacted.
- HMAC, Stripe, GitHub, and Shopify resign replay refresh provider-valid signatures where supported.
- Generated Pest-compatible tests load fixtures through relative paths where possible.
- Sensitive fixture payloads are written under `.local` and gitignored when the fixture root is inside a git repo.
- Live tunnel capture requires both `--live` and `preview.live_enabled=true`.
- Route preview is covered for named route lookup, required/optional/domain parameter handling, TTL signed links, proxied GET execution, expired/tampered signature rejection, non-GET/HEAD blocking, guard summary output, app-specific user context, fake flag behavior for supported Laravel facades including mail, and `--readonly-db` rollback for covered DB writes.
- Clean Laravel 12.58.0 consumer install passed through a Composer path repository. `php artisan package:discover` discovered `oxhq/preview`, all preview commands were listed, synthetic generic capture created `20260505011852323-sugxujb2`, and `preview:capture:list` read it back from the consumer app storage path.

## Proof Boundary

This is package-internal Testbench proof plus a Composer path-repository Laravel 12 consumer smoke. It is not hosted CI, package publication, Packagist installation, or live tunnel startup with real cloudflared/ngrok binaries. Pest-compatible generated files are executed through the package acceptance shim because Pest is not a dev dependency of this package.

Route preview proof now includes signed proxy execution, session context carried into proxied preview requests, app-specific authenticated preview through `--user-id` plus an authenticatable model, supported fake flag behavior, and database rollback for covered writes. It does not prove generic authorization bypass, policy bypass, cache/filesystem isolation, live external route sharing, or complete side-effect isolation beyond the explicit fake flags covered by package tests.

Scenario workbench proof is outside v0.1. The current v1.0 proof includes local PHP scenario files, read-only scenario list/show behavior, seed execution through Laravel's normal seeder path, and capture replay composition through `preview:scenario:replay {scenario}` with exact/resign modes. It does not prove route composition or scenario test generation until those commands and tests exist.

Scenario proof now includes:

- seed composition proof that the configured seeder runs through Laravel's normal seeder path before replay and fails clearly on seeder errors
- capture replay composition proof that `preview:scenario:replay {scenario} --exact` replays stored raw bodies and captured headers for each listed capture
- capture replay composition proof that `preview:scenario:replay {scenario} --resign` replays stored raw bodies with fresh provider-valid signature headers where supported and fails clearly where unsupported
- optional HTTP dispatch proof that `preview:scenario:replay {scenario} --exact --send-to={url}` sends each listed capture through the shared replay dispatcher
- route composition proof only after named route previews execute under the documented route-preview safety boundaries
- scenario test generation proof only after Pest-compatible scenario tests are generated and behavior-tested
