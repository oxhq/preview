# v0.3 Safe Route Preview Plan

## Target Command

```bash
php artisan preview:route {route} --ttl=2h --param=id=123 --readonly-db --guard=client
```

## First Slice

The first v0.3 slice should make route previews inspectable and shareable without pretending that the package can safely execute arbitrary application behavior yet.

Ship:

- named route lookup by `{route}`
- route method, URI, action, domain, and middleware summary
- route params from repeated `--param=key=value` flags
- TTL signed access link generation
- optional guard context summary from `--guard`
- default blocking for routes that do not allow `GET` or `HEAD`
- a future explicit-allow path for non-`GET`/`HEAD` routes
- `--readonly-db` accepted as a declared safety flag with warnings

Do not ship in this slice:

- full request transaction rollback
- proxy execution through the signed link
- queue, mail, event, HTTP, cache, or filesystem fakes
- automatic guard/session impersonation beyond summary output
- default exposure of `POST`, `PUT`, `PATCH`, or `DELETE` routes

## Safety Contract

`--readonly-db` is the flag name. Do not introduce `--readonly`.

`--readonly-db` is not full readonly. For this slice, it means the generated preview output records the operator's intent and warns about unsupported side effects. It does not prove that route execution cannot mutate application state.

Database rollback, when added later, will only cover database writes inside the wrapped request transaction. It will not protect:

- queued jobs
- sent mail
- cache writes
- filesystem writes
- external HTTP calls
- emitted events with side effects
- code that uses separate database connections outside the wrapper

## Implementation Notes

1. Resolve `{route}` through Laravel's named route collection.
2. Fail clearly when the named route does not exist.
3. Render a middleware summary before printing any access link.
4. Require all mandatory route parameters to be provided through `--param`.
5. Sign generated access links with the requested TTL.
6. Block routes that do not advertise `GET` or `HEAD` unless a later explicit-allow option is implemented.
7. Print `--readonly-db` warnings whenever the flag is present.

## Follow-Up Work

Later v0.3 work can add proxied execution, request transaction rollback, explicit unsafe-method opt-in, guard/session impersonation, and side-effect fakes. Those follow-ups should remain separate from this first metadata-and-link slice so the initial command has a truthful safety boundary.
